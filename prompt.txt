AESTHETIC DIRECTION — READ THIS FIRST
================================================================================

This is THE most important section. Everything else follows from this.

THE LOOK AND FEEL:

Think: a cross between a bold editorial magazine, a premium product launch
page, and a clean modern SaaS site. The design should feel ALIVE and CONFIDENT
— not quiet and brooding, not dark and cinematic. It should feel like opening
a beautifully designed product box.

CORE AESTHETIC PILLARS:

1. BOLD, OVERSIZED SANS-SERIF TYPOGRAPHY
   — Headlines are MASSIVE. Think text-7xl, text-8xl, even text-9xl on desktop.
   — Font weight is EXTRABOLD (800) or BLACK (900) for display headings.
   — Body text is clean, medium-weight sans-serif.
   — NO SERIF FONTS ANYWHERE. No Bodoni, no Playfair, no Garamond, no Times,
     no Georgia, no Fraunces, no Instrument Serif. Zero serifs on the entire
     site.
   — Recommended sans-serif families (pick ONE display + ONE body):
     Display: Outfit, Cabinet Grotesk, Satoshi, PP Neue Montreal, Syne,
              Clash Display, General Sans, Switzer
     Body: Geist, Inter, DM Sans, Plus Jakarta Sans, Manrope
   — Letter-spacing on headlines: very tight (tracking-tighter or -0.04em).
   — Letter-spacing on small labels/nav: very wide (tracking-widest or 0.2em),
     uppercase.
   — Headlines should feel like they OWN the page. Massive, bold, tight.

2. CLEAN LIGHT BACKGROUND + BOLD GEOMETRIC YELLOW ACCENT
   — The base background is LIGHT — a clean off-white or very pale warm gray.
     Not dark. Not black. Think #FAFAFA, #F5F5F5, or similar.
   — The primary accent is a confident YELLOW — specifically around the
     yellow-400/amber-400 range (#FACC15, #FBBF24, #EAB308). A warm, saturated,
     unapologetic yellow.
   — Yellow is used as a BOLD GEOMETRIC SHAPE — not as a subtle tint. Think:
     a large filled circle behind a product image, a solid yellow background
     block, a thick yellow underline on a headline, a yellow-filled card.
   — Yellow is the ONLY accent color. Nothing else. No gradients mixing yellow
     with orange or green. Pure yellow.
   — Do NOT use yellow for body text (unreadable on light backgrounds). Yellow
     is for shapes, backgrounds, buttons, badges, and decorative elements.

3. TEXT COLOR HIERARCHY VIA OPACITY
   — Instead of using different gray hex values for text hierarchy, use
     OPACITY on the foreground color:
     - Primary text: text-foreground (full opacity, near-black #0A0A0A or
       #171717)
     - Secondary text: text-foreground/80
     - Tertiary/muted text: text-foreground/60
     - Placeholder/hint text: text-foreground/40
     - Ghost text: text-foreground/20
   — This creates a cohesive, unified palette where everything is the same
     hue at different intensities. Very clean.

4. MUTED SURFACE LAYERS
   — Cards and containers use very soft muted backgrounds, not hard borders:
     bg-muted (a soft warm gray like #F0F0F0 or #E8E8E8)
     bg-muted/50 (half-opacity muted)
   — Borders are minimal. When used, they are very faint: border-foreground/10
     or border-foreground/5.
   — Shadows are soft and diffused, not dramatic. Think:
     shadow-[0_2px_8px_rgba(0,0,0,0.04)] for cards
     shadow-[0_8px_30px_rgba(0,0,0,0.08)] for elevated elements
   — Rounded corners: generous but not pill. Default radius: rounded-2xl
     (16px) for cards, rounded-xl (12px) for buttons and inputs.

5. SPRING-PHYSICS MICRO-INTERACTIONS
   — All hover and interaction animations use SPRING physics, not linear
     easing. Specifically: { type: "spring", stiffness: 400-600, damping: 25-35, mass: 0.5 }
   — Buttons: on hover, subtle scale(1.02) + slight shadow increase. On press:
     scale(0.97) for tactile push feel.
   — Cards: on hover, a very subtle lift (translateY: -2px) with spring.
   — Inputs: custom animated caret with spring physics. Focus states use a
     soft outline ring, not a harsh border change.
   — Navigation links: smooth opacity transitions on hover (0.6 → 1.0).
   — These micro-interactions should feel PHYSICAL and RESPONSIVE — like
     touching real objects. Not floaty, not slidey. Snappy springs.

6. 3D SCROLL PERSPECTIVE EFFECTS
   — Key sections use scroll-driven 3D perspective transformations:
   — The featured product section should have a "floating card" effect where
     a card containing the product starts rotated in perspective (rotateX: 20deg)
     and flattens to 0 as the user scrolls. Scale starts at 1.05 and settles
     to 1.0.
   — This creates a dramatic reveal moment as you scroll.
   — Use CSS perspective: 1000px on the parent container.
   — Heavy layered box shadows on the floating card for depth:
     "0 0 #0000004d, 0 9px 20px #0000004a, 0 37px 37px #00000042,
      0 84px 50px #00000026, 0 149px 60px #0000000a, 0 233px 65px #00000003"
   — The card has a thick border (3-4px, medium gray like #6C6C6C or
     foreground/20), large border-radius (30px), and a slightly dark inner
     background (#222222 or similar) that contrasts with the light page.

7. SPATIAL LAYOUT — NOT CENTERED-EVERYTHING
   — The hero is NOT a centered headline over a gradient. It uses a 3-COLUMN
     GRID where:
     - Left column: short body text + "Read more" link
     - Center column: product image floating over a large yellow circle
     - Right column: MASSIVE oversized headline (text-8xl / text-9xl)
   — This creates an asymmetric, editorial, magazine-spread feel.
   — On mobile, this collapses to: headline on top → product image with
     yellow circle → body text below.
   — Other sections can use split layouts, but vary the composition. Not
     every section should be two-equal-columns.

================================================================================
MOTION RULES
================================================================================

1. ALL animations use spring physics, not linear or ease-in-out:
   Default spring: { type: "spring", stiffness: 500, damping: 30, mass: 0.5 }

2. Page entry animations: elements fade in (opacity 0→1) with a slight
   translateY (20px → 0), staggered by 0.1-0.2s per element.

3. Scroll-driven animations: the 3D card section uses useScroll + useTransform
   for perspective rotation. No other scroll-hijacking.

4. Hover: spring-based scale and shadow transitions.

5. Reduced motion: ALL animations are disabled when the user has
   prefers-reduced-motion enabled. Elements appear immediately, no transforms,
   no springs.

6. No infinite loops except the marquee. No decorative pulsing. No ambient
   floating. Motion is motivated and purposeful.


================================================================================
ACCESSIBILITY
================================================================================

— Visible focus indicators on all interactive elements (a soft outline ring,
  outline-foreground/20, 2px offset)
— Skip-to-content link (hidden, visible on focus)
— All images: descriptive alt text
— Form inputs: proper labels (not placeholder-as-label)
— Color contrast: WCAG AA minimum. Yellow buttons get dark text (foreground,
  not white) because yellow-on-white fails contrast.
— Semantic HTML: nav, main, article, section, header, footer
— ARIA labels on hamburger menu, dropdowns
— Error messages: inline, clear, no alert() dialogs


================================================================================
WHAT TO AVOID — HARD NOs
================================================================================

— NO serif fonts. Zero. None. Not even for "one accent headline."
— NO dark mode for public pages. This is a LIGHT site with bold yellow accent.
— NO gold/bronze cinematic dark theme. That was the old design. Kill it.
— NO conic gradients, specimen rings, film grain overlays, atmospheric fixed
  backgrounds. Those are all from the old design.
— NO glass morphism / frosted glass / backdrop-blur on cards. Clean muted
  solids instead.
— NO "01 / 02 / 03" numbering on things that aren't real sequences (but DO
  number the process pipeline because that IS a real sequence).
— NO purple, blue, or green accents. Yellow only.
— NO gradient blobs or mesh gradients.
— NO AI-purple glow aesthetic.
— NO over-animation. Motion is purposeful, spring-based, motivated.
— NO placeholder/Lorem Ipsum text. Use real content.
— NO generic stock supplement photos. Use the actual product image.
— NO "elegant dark premium" aesthetic. This is BOLD, BRIGHT, CONFIDENT.
